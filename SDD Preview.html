1.1 Design Goals

Present Ophelia's professional identity, academic growth, and ACPA/NASPA competencies in a clean, compelling, and navigable site. Target the Exemplary tier across all rubric criteria (200/200 pts). The site must be fully usable by visitors with disabilities — including screen reader users, keyboard-only navigators, and users with low vision — and require zero external accounts or hosting services to open and evaluate.

4.1 Shared Layout Structure

Every page follows this consistent structure, ensuring predictable navigation and semantic landmarks for assistive technology:

6 Accessibility Specification — WCAG 2.1 AA

6.1 Skip Navigation Link

The "Skip to main content" link is the first focusable element on every page. It is visually hidden until a keyboard user focuses it.

Live example — press Tab to see it appear:
Skip to main content

Normally hidden off-screen; appears on keyboard focus. Target: #main

6.2 Semantic HTML Structure

ElementRequired Usage
<header>Site-wide branding + navigation (one per page)
<nav aria-label>Primary navbar, labeled for screen readers
<main id="main">Primary page content — skip link target
<section aria-labelledby>Each thematic section, linked to its heading
<article>Individual competency blocks
<footer>Site-wide contact info + copyright
<button>All interactive triggers — never <div> or <span>

6.3 Images & Alt Text Rules

Image TypeRule
Informative imagesalt="[descriptive text]" required
Decorative imagesalt="" (empty) — screen readers skip
Profile photoalt="Professional photo of Ophelia Ivy Mensah"
Icons (decorative)aria-hidden="true" + visible label
Icons (standalone)role="img" + aria-label="[purpose]"
No images of textAll text must be real HTML — not baked into images

6.4 Keyboard Navigation

InteractionKey
Navigate links / buttonsTab / Shift+Tab
Activate buttons / linksEnter or Space
Open / close mobile navEnter on hamburger button
Open / close accordionEnter or Space
Dismiss any overlayEscape (returns focus to trigger)
CSS /* Visible focus ring for keyboard users */ :focus-visible { outline: 3px solid #1A5E9C; outline-offset: 3px; border-radius: 2px; } /* Remove outline for mouse users */ :focus:not(:focus-visible) { outline: none; }

6.5 High-Contrast Mode Toggle

A button in the site header lets users switch to high-contrast mode (dark background, high-contrast text). Preference persists in localStorage.

Example — toggle preview:
CSS [data-theme="high-contrast"] { --color-bg: #000000; --color-text: #FFFFFF; --color-heading: #FFFF00; --color-muted: #DDDDDD; --color-link: #66CCFF; }

6.6 Accessible Form Design

HTML <div class="form-group"> <label for="name"> Full Name <span aria-hidden="true">*</span> <span class="sr-only">(required)</span> </label> <input id="name" type="text" required aria-required="true" autocomplete="name"/> </div>

6.7 Accordion ARIA Pattern

HTML <button aria-expanded="false" aria-controls="panel-as" id="btn-as"> Advising & Supporting (AS) </button> <div id="panel-as" role="region" aria-labelledby="btn-as" hidden> <!-- competency content --> </div>

6.8 Reduced Motion

CSS @media (prefers-reduced-motion: reduce) { *, *::before, *::after { animation-duration: 0.01ms !important; animation-iteration-count: 1 !important; transition-duration: 0.01ms !important; scroll-behavior: auto !important; } }

6.9 Screen Reader Utility Class

CSS /* Visually hidden but announced by screen readers */ .sr-only { position: absolute; width: 1px; height: 1px; padding: 0; margin: -1px; overflow: hidden; clip: rect(0,0,0,0); white-space:nowrap; border: 0; }

6.10 Accessible Artifact Download Links

HTML <!-- ✅ Accessible — descriptive label with format + size --> <a href="assets/artifacts/equity-audit.pdf" download aria-label="Download Final Equity Audit — PDF"> <i class="fa-solid fa-file-pdf" aria-hidden="true"></i> Download Equity Audit (PDF) </a> <!-- ❌ Inaccessible — avoid "click here" links --> <a href="#">Click here</a>

10 JavaScript Interactions

10.1 Mobile Nav Toggle

JavaScript const menuBtn = document.getElementById('menu-toggle'); const mobileMenu = document.getElementById('mobile-menu'); menuBtn.addEventListener('click', () => { const isOpen = menuBtn.getAttribute('aria-expanded') === 'true'; menuBtn.setAttribute('aria-expanded', String(!isOpen)); mobileMenu.hidden = isOpen; }); // Close on Escape — return focus to trigger document.addEventListener('keydown', (e) => { if (e.key === 'Escape' && !mobileMenu.hidden) { mobileMenu.hidden = true; menuBtn.setAttribute('aria-expanded', 'false'); menuBtn.focus(); // Critical: return focus to opener } });

10.2 Accordion

JavaScript document.querySelectorAll('.accordion-trigger').forEach(trigger => { trigger.addEventListener('click', () => { const expanded = trigger.getAttribute('aria-expanded') === 'true'; const panel = document.getElementById( trigger.getAttribute('aria-controls') ); trigger.setAttribute('aria-expanded', String(!expanded)); panel.hidden = expanded; }); });

12 Testing Checklist

Accessibility Testing

  • Run axe DevTools browser extension — zero critical violations
  • Keyboard-only navigation — all pages fully functional without mouse
  • Screen reader test (NVDA + Firefox or VoiceOver + Safari)
  • Zoom to 200% — no content loss, no horizontal overflow
  • Resize browser to 320px width — reflow test
  • Activate high-contrast toggle — all text remains readable
  • Enable OS "Reduce Motion" — no animations play

Content & Links

  • All internal page links work
  • All artifact download links present (placeholders acceptable)
  • Resume PDF link opens correctly
  • LinkedIn URL correct
  • Spell-check all text content

Cross-Browser

  • Chrome (latest)
  • Safari (latest)
  • Firefox (latest)
  • Mobile Safari (iOS)
  • Chrome Mobile (Android)

Rubric Verification

  • Resume present and complete ✓
  • Mission statement prominently displayed ✓
  • 3 Exemplary competencies with narratives + artifacts (AS, SJI, LEAD) ✓
  • 3 Proficient competencies with 2+ activity descriptions (AER, PEF, SLD) ✓
  • 3 Foundational competencies present (VPH, LPG, OHR) ✓
  • Design exceeds basic template — custom palette, typography, accessibility ✓
  • All links functional ✓
  • Grammar and proofreading complete ✓
